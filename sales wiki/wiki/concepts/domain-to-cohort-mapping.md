## Definition

The framework for mapping any prospect's domain/role to specific cohort modules and concrete outcomes. The #1 deal-killer identified from real calls: rep can't answer "what can I specifically get from this cohort in MY field?" This concept provides the thinking framework to answer that question for any domain.

## Why It Matters (for sales)

This is the most critical tool for mid-call. When a cybersecurity engineer asks "what can I get out of this?" and the rep can't map curriculum → their domain specifically, the lead leaves unconfident. This framework + web search fills that gap for ANY domain that walks in.

## How to Use It

**Step 1:** Identify their domain clearly (from discovery)

**Step 2:** Map to the 4 cohort modules:

| Module | Best fit domains |
|--------|-----------------|
| **Diffusion** (images, video) | Designers, animators, filmmakers, game designers, architects, photographers, marketing (visual content), media agencies |
| **Full Stack** (apps, workflows, no-code) | Founders, managers, operations professionals, anyone wanting to build internal tools, non-coders especially |
| **LLMs** (language models, text, documents) | Lawyers, doctors, researchers, journalists, educators, analysts, HR, customer-facing roles, content creators |
| **Agents** (automation, autonomous workflows) | Engineers, operations, logistics, founders wanting to automate business functions, anyone with repetitive multi-step processes |

**Step 3:** Identify the specific use case within their domain. Ask yourself:
- What is repetitive and slow in this domain? → Agents/Full Stack automates it
- What requires synthesis of large amounts of text? → LLMs handles it
- What visual/creative work could be accelerated? → Diffusion module
- What new service could they offer clients? → Any module

**Step 4:** Combine with web search for current AI trends in their field (Claude should search live for domains not covered in wiki)

## Examples by Domain

**Cybersecurity engineer:**
- LLMs → build threat intelligence parsers, CVE summarization, incident report generation
- Agents → automate SOC Tier-1 alert triage (90% automation is real in 2026)
- Full Stack → build internal security dashboards, vulnerability tracking tools

**Media agency founder:**
- Diffusion → AI video generation, AI presenters, automated B-roll creation
- Agents → automate content pipeline end-to-end (storyboard → script → video → publish)
- Full Stack → client reporting automation

**Designer / UX:**
- Diffusion → rapid design variations at scale, mood boards, prototype visuals
- Full Stack → build AI-assisted design tools, automate design feedback pipelines
- LLMs → user research synthesis, UX copy generation, accessibility automation

**Marketer:**
- LLMs → campaign copy, SEO content at scale, competitor analysis automation
- Agents → multi-channel campaign orchestration, lead scoring automation
- Full Stack → custom analytics dashboards, automated reporting

**Senior manager / executive:**
- LLMs → synthesize team reports, board presentations, strategy documents
- Agents → automate reporting pipelines, build internal knowledge bases
- Full Stack → executive decision support tools

**Startup founder:**
- Any module → build AI features into their product
- Full Stack → build MVP AI products faster
- Agents → automate operations, customer support, onboarding

**Doctor / medical professional:**
- LLMs → clinical note summarization, diagnostic cross-referencing, medical literature review
- Agents → patient communication automation, appointment optimization

**Lawyer:**
- LLMs → contract review, legal research automation, document generation
- Agents → due diligence automation, case file summarization

**Game designer / animator:**
- Diffusion → asset generation, environment creation, character variations
- Agents → procedural content generation, NPC dialogue systems

**Teacher / educator:**
- LLMs → personalized curriculum generation, student feedback at scale
- Full Stack → build custom learning tools, AI tutors for students

## The Universal Formula

Whatever their domain:
> [Their domain expertise] + [relevant cohort module] = [specific thing they can build or automate in their world]

Always make it specific. "AI will help you" converts nobody. "The agents module will let you automate [specific repetitive task they just told you about]" converts.
